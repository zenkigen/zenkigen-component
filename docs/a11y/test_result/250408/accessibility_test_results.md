# コンポーネントアクセシビリティテスト結果

このドキュメントは、各コンポーネントのアクセシビリティテスト結果をまとめたものです。

## テスト概要

- テスト環境: Storybook (http://localhost:60599/)
- テストツール: Playwright
- テスト対象: 全コンポーネント

## テスト結果

各コンポーネントのアクセシビリティ問題を以下に記載します。

- キーボード操作: ✅ (表示のみのコンポーネントのため該当なし)
- コントラスト比: ✅
- テキスト代替: ✅

#### 検出された問題

- 特に問題は検出されませんでした。

#### 備考

- ユーザーIDと名前に基づいて適切に表示されています。
- サイズのバリエーション（x-small, small, medium, large, x-large）が適切に実装されています。
- isDisabledプロパティが適切に機能しています。

### 2. Breadcrumb

- キーボード操作: ✅ (すべてのリンクにキーボードでアクセス可能)
- コントラスト比: ✅
- フォーカス可視性: ✅

#### 検出された問題

- 特に問題は検出されませんでした。

#### 備考

- 現在のページは適切にアクティブ表示され、リンクではなくテキストとして実装されています。
- セパレーター（/）が視覚的に表示されていますが、スクリーンリーダーでは適切に処理されます。

### 3. Button

#### 確認事項

- 適切なARIAロール: ✅ (ネイティブの `button` 要素を使用)
- キーボード操作: ✅ (キーボードでアクセス可能、Enterキーで操作可能)
- コントラスト比: ✅ (すべてのバリエーションで十分なコントラスト)
- フォーカス可視性: ✅

#### 検出された問題

- 特に問題は検出されませんでした。

#### 備考

- 適切な状態変化（isSelected, isDisabled）が実装されています。
- 様々なバリエーション（fill, fillDanger, outline, text）が提供されており、すべて適切にアクセシブルです。
- サイズ（small, medium, large）のバリエーションが提供されています。
- カスタムの borderRadius を設定する機能があります。
- アイコンの追加（before, after）ができますが、純粋に装飾目的の場合は適切に処理されています。

### 4. Checkbox

#### 確認事項

- 適切なARIAロール: ✅ (ネイティブの `checkbox` 要素を使用)
- キーボード操作: ✅ (Tabキーでフォーカス、Spaceキーで選択・解除可能)
- コントラスト比: ✅ (状態の違いが視覚的に明確)
- ラベルとの関連付け: ✅ (適切に関連付けられています)

#### 検出された問題

- 特に問題は検出されませんでした。

#### 備考

- 三つの状態（未選択、選択済み、不確定）が適切に実装されています。
- 適切な状態変化（isChecked, isIndeterminate, isDisabled）が実装されています。
- 色のバリエーション（default, gray, error）が提供されており、エラー状態の視覚的フィードバックが適切です。
- 適切なラベルが関連付けられており、クリック可能な領域が十分に確保されています。
- id, name, value 属性が適切に設定可能で、フォーム送信時の挙動が考慮されています。

### 5. Dropdown

#### 確認事項

- 適切なARIAロール: ✅ (トリガーボタンと展開メニューの適切なロール)
- キーボード操作: ✅ (Tabキーでフォーカス、Enter/Spaceキーで開閉、矢印キーでメニュー内移動が可能)
- コントラスト比: ✅
- フォーカス可視性: ✅
- 展開状態の伝達: ✅ (アイコン方向の変更で状態を視覚的に示しています)

#### 検出された問題

- 🟡 アイコンに適切な代替テキストが設定されておらず、スクリーンリーダーでは「angleSmallDown」「angleSmallUp」と読み上げられる可能性があります。

#### 備考

- ドロップダウンメニューが適切な `list` および `listitem` 構造で実装されています。
- さまざまなサイズ（x-small, small, medium, large）とスタイル（text, outline）のバリエーションが提供されています。
- アイコンの表示・非表示を制御できる `isArrowHidden` プロパティがあります。
- `isDisabled` 状態が適切に実装されています。
- Portalを使った実装オプションもあり、複雑なレイアウトでも適切に表示できます。
- 改善推奨: アイコンには適切な代替テキスト（「メニューを開く」「メニューを閉じる」など）を設定することが望ましいです。

### 6. TextInput

#### 確認事項

- 適切なARIAロール: ✅ (ネイティブの `textbox` 要素を使用)
- キーボード操作: ✅ (キーボードでアクセス可能、テキスト入力が可能)
- コントラスト比: ✅
- プレースホルダーテキスト: ✅
- エラー状態の視覚的フィードバック: ✅

#### 検出された問題

- 🟡 ラベルが明示的に関連付けられていない可能性があります。プレースホルダーテキストのみでは不十分な場合があります。

#### 備考

- サイズバリエーション（medium, large）が適切に実装されています。
- エラー状態（isError）が視覚的に明確に表示されます。
- 無効状態（disabled）が適切に実装されています。
- クリアボタン機能（onClickClearButton）がサポートされています。
- 改善推奨: 入力フィールドには常に明示的にラベルを関連付けることが望ましいです。特にプレースホルダーテキストは入力開始時に消えるため、スクリーンリーダーユーザーや認知障害のあるユーザーにとって問題となる可能性があります。

### 7. Modal

#### 確認事項

- 適切なARIAロール: ✅ (モーダルダイアログとして適切なロール)
- キーボード操作: ✅ (フォーカストラップの実装、Escキーでの閉じる機能)
- コントラスト比: ✅
- 閉じるボタン: ✅ (明確に識別可能)

#### 検出された問題

- 🟡 モーダル開閉時のフォーカス管理が最適化されていない可能性があります。
- 🟡 閉じるボタンのアイコンに適切な代替テキストが設定されていない可能性があります。

#### 備考

- モーダルの表示・非表示の切り替えが適切に実装されています。
- オーバーレイによる背景の暗転が適切に実装されています。
- 適切なボタン配置（キャンセル・保存）が実装されています。
- 幅の設定が可能です（例: 480px）。
- 様々なバリエーション（チェックボックス付き、サブボタン付き、固定高さ、タブ付き、ボタンなし、危険操作用）が提供されています。
- 改善推奨: モーダル開閉時のフォーカス管理を最適化し、閉じるボタンには適切な代替テキスト（「閉じる」など）を設定することが望ましいです。

## 総合評価

### 良い点

- ほとんどのコンポーネントが適切なネイティブHTML要素を使用しており、適切なARIAロールが実装されています。
- キーボード操作が全般的に考慮されています。
- すべてのコンポーネントでコントラスト比が適切に確保されています。
- 状態変化（無効、エラー、選択中など）が視覚的に明確に示されています。
- コンポーネントのサイズやバリエーションが適切に提供されており、様々なユースケースに対応できます。

### 改善が必要な点

- アイコンの代替テキストが不適切または欠落している場合があります（Dropdown、Modal）。
- ラベルの関連付けが明示的でない場合があります（TextInput）。
- モーダルのフォーカス管理が最適化されていない可能性があります。

### 今後の推奨事項

1. すべてのアイコンに適切な代替テキストを設定する。
2. すべての入力フィールドに明示的なラベルを関連付ける。
3. モーダルやドロップダウンなどの複雑なコンポーネントでは、キーボードフォーカスの管理を最適化する。
4. スクリーンリーダーでのテストを定期的に実施し、非視覚的なユーザーエクスペリエンスを確認する。
5. アクセシビリティガイドラインをコンポーネントライブラリのドキュメントに追加し、開発者がアクセシブルな実装方法を理解できるようにする。
