## 概要

リスト形式の複数の選択肢の中から選択できるコンポーネントです。Listと組み合わせて使用します。

## 設計の原則 （Do）

### 選択肢が4つ以上ある場合に使用します

選択肢が4つ以下の場合は、代わりにRadioButtonの使用を検討してください。

### 選択肢は論理的な順序で並べます

ユーザーが目的の項目を簡単に見つけられるよう、五十音順、使用頻度順など選択肢の並び順には一貫したルールを設けてください。

## 推奨しないこと（Don't）

### 二者択一の時に、Selectが最適というケースは基本的にない

ただし、Selectの中の選択肢の数が状況に応じて変動するケースで場合によっては選択肢が2つになることはあります。また、表などのスペースの制約が強い時に、二者択一の選択をあえてSelectにするという可能性はあります。
![推奨しないこと その1](/storybook/images/select_01.jpg)

### 選択肢が2つの時は基本的に使用しない

選択肢が2つで更にはっきり示せる場合(Yes/Noなど)は1つのCheckBox、または、Toggleにまとめます。ただし、Selectの中の選択肢の数が状況に応じて変動するケースで場合によっては選択肢が2つになることはあります。また、表などのスペースの制約が強い時に、二者択一の選択をあえてSelectにするという可能性はあります。

### 選択しただけでアクションをトリガーしない

ユーザーがドロップダウンから項目を選択した瞬間に、フォームの送信やページの再読み込みといったアクションを自動で実行しないでください。アクションの実行は「決定」や「適用」といった**明確なボタン**によって、ユーザーの意思で行われるべきです。
