## 概要

Toastコンポーネントは、画面に短時間表示される小さなメッセージを表示するUI要素です。
ユーザーにフィードバックを提供したり、簡単な情報を伝えたりする際に役立ちます。

## 設計の原則（Do）

### ユーザーに成功、警告、エラーなどのフィードバックを提供します

メッセージは画面左下に、ユーザーの操作の邪魔にならないコンポーネントとして表示されます。

### 連続して使用する場合は時系列順に並べます

トーストを連続して複数表示する場合は、古いものが下、新しいものがその上に、順番に縦に積み上げて表示します。

### 内容に合わせて適切なステートを使用します

成功、警告、エラー、情報など内容に合わせて適切なステートを選択してください。

### エラーを通知する場合はユーザーに具体的な解決策を示してください

エラーを通知する場合は、ユーザーが解決のためにとるべき行動を一緒に提示してください。
![設計の原則 その4](/storybook/images/toast_01.jpg)

### できる限り短く、説明的で汎用性のある言葉を使用してください

・専門すぎる用語(社内でしか使わない単語)や難解な言葉を使用するのは極力避けてください。
・ショックや不安を煽る文章は使わないでください。
・できる限り丁寧で優しく、好印象を与える表現を使用してください。
![設計の原則 その5](/storybook/images/toast_02.jpg)

## 推奨しないこと（Don’t）

### 長すぎるメッセージを設定しない

一文〜二文で簡潔にライティングします。
![推奨しないこと その1](/storybook/images/toast_03.jpg)

### ユーザーを非難しない

口調や言葉選びは、ユーザーのメッセージの理解に多大な影響を及ぼします。適切な語り口を使って伝えれば、ユーザーがエラーの起きた状況に共感しやすくなり、エラーの理解が容易になります。
![推奨しないこと その2](/storybook/images/toast_04.jpg)
