## 概要

ページや要素単位における重要度が高い内容を、小さなメッセージで表示するUIコンポーネントです。ユーザーの対応が必要な情報を通知してアクションさせたい際に役立ちます。

## 設計の原則（Do）

### ユーザーに成功、警告、エラーなどのフィードバックを提供します

画面ファーストビューに配置し、メインコンテンツを邪魔しないように且つユーザーが閉じるまで表示されます。

### 連続して使用する場合は時系列順に並べます

複数のメッセージが表示される場合は新しい順に並びます。

### エラーを通知する場合はユーザーに具体的な解決策を示してください

メッセージを閉じるアクションが必要となり、多用するとわずらわしいUXとなるので、重要な状況に限って使用することを推奨します。

エラーを通知する場合は、ユーザーが解決のためにとるべき行動を一緒に提示してください。
![設計の原則 その3](/storybook/images/notification-inline_01.jpg)

### できる限り短く、説明的で汎用性のある言葉を使用してください

・専門すぎる用語(社内でしか使わない単語)や難解な言葉を使用するのは極力避けてください。
・ショックや不安を煽る文章は使わないでください。
・できる限り丁寧で優しく、好印象を与える表現を使用してください。
![設計の原則 その4](/storybook/images/notification-inline_02.jpg)

# 推奨しないこと（Don’t）

### 長すぎるメッセージを設定しない

一文〜二文で簡潔にライティングします。
![推奨しないこと その1](/storybook/images/notification-inline_03.jpg)

### ユーザーを非難しない

エラーメッセージは「行われた操作のため」などのユーザーのアクションではなく、問題そのものに焦点を当てるよう心掛けましょう。
![推奨しないこと その2](/storybook/images/notification-inline_04.jpg)
