# Introduction

## デザインシステムとは？

デザインシステムとは、デジタルプロダクトの目的を達成するために首尾一貫したルールで編成された、お互いに関連付けられたパターンとその実践方法である。

プロダクトを競合他社と差別化するのはパターンの目新しさではなく、これたのパターンがどのように実行され、応用されているか、そしてどのように連携してデザインの目的を達成しているか。

- [デジタルプロダクトのためのデザインシステム実践ガイド](https://amzn.to/3FYZ7W0)

## なぜデザインシステムを成長させる？

> デザインシステムの成長は、単なる効率化の手段ではなく、ユーザーにとっての最高の体験を実現するための基盤である。これにより、プロダクトの品質を向上させ、持続的な価値を提供することができる。

### 1. 車輪の再発明の回避と時間の有効活用

デザイナーやエンジニアがゼロから設計する手間を省き、一貫性のあるデザインを迅速に実装可能にする。

- UIの品質が安定し、情報設計やUX改善の時間に費やせる。
- エンジニアとの共通言語が生まれ、スムーズな開発を可能にする。
- WCAG2.1に準拠したアクセシビリティが標準化を目標とする。
- プロトタイピングが容易になり、アイデアの具現化が加速する。

### 2. ブランドとしての信頼性向上

統一されたデザインとアクセシビリティの基準を持つことで、企業の姿勢を明確にし、信頼を築く。

- ユニバーサルデザインのフォントを採用し、視認性と統一感を確保。
- WCAG2.1準拠を推進し、ユーザーフレンドリーなプロダクトを目指す。
- 企業のデザインに対する姿勢を明確にし、共感を生む。

### 3. デザインをデザイナーのものだけにしない

デザイナーが介在しなくても、誰もが直感的に使いやすいUIを作成できる環境を整備する。

- 企画者やエンジニアが迅速にプロトタイプを作る事ができる。
- デザインシステムをプロダクトとして捉え、使いやすさの指標を明確にする。
- UI/UXの品質を競争力とし、事業成長に貢献する。

### デザインシステムは手段であり、目的ではない

デザインシステムの本質は、効率化や一貫性の確保だけではない。最も大切なのは、それを通じてユーザーにどれだけ価値を提供できるかである。強固なデザインシステムを築くことで、プロダクトの成長とともに企業の競争力をも高めていくことができる。
