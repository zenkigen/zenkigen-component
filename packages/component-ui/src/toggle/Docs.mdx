## 概要

Toggleでは、ユーザーは相互排他的な2つの選択肢の中からどちらかを選ぶことになり、必ず初期値があります。
Toggleは結果を即座に提供し、選択を随時コントロールできるようにしなければなりません。

# 設計の原則（Do）

### ToggleをON/OFFすると、直ちに関連する要素へ影響します

Toggleは、照明のスイッチのように、すぐに効果を発揮するものでなければなりません。ステータスをすぐに変更するために使用します。

### トグルスイッチの条件

１. 利用環境ではなくシステムの状態を有効化する

２. 対立的選択肢ではなく並列的選択肢を示している

## 推奨しないこと（Don’t）

### 別のボタンのせいで反映が遅れるとすれば、Toggleは使わない

トグルを操作したあと、即座に反応せず、サブミットが必要な場合は使用しないでください。選択した後にサブミットを使用する場合はCheckboxを使用してください。
![推奨しないこと その1](/storybook/images/toggle_01.jpg)

### 対立する選択には使用しない

Toggleは0か1の選択のためのもので、対立する選択のためのものではありません。異なる状態を表す場合は[SegmentedControl](/docs/components-segmentedcontrol--docs)を使用します。
![推奨しないこと その2](/storybook/images/toggle_02.jpg)

### 操作する対象をラベルにする

トグルのUI自体がON/OFFの状態を表している為、ラベルには操作対象を明示してください。
![推奨しないこと その3](/storybook/images/toggle_03.jpg)
